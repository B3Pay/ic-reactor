IC Reactor is designed to be highly compatible with AI coding assistants (like ChatGPT, Claude, and GitHub Copilot) and LLMs. This guide explains how to get the best results when using AI to build with IC Reactor.

## AI Context Files

The repository keeps AI-facing context in a small number of canonical files:

| File                               | Best for                                                                                                            |
| ---------------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| `/llms.txt`                        | Compact package ownership, task routing, generated-file boundaries, and verification commands                       |
| `/llms-full.txt`                   | Longer copy-paste guide with setup patterns, inside/outside React guidance, auth, codegen, and dynamic Candid notes |
| `/AGENTS.md`                       | OpenAI Codex instructions for this repository                                                                       |
| `/CLAUDE.md`                       | Claude project context                                                                                              |
| `/.github/copilot-instructions.md` | GitHub Copilot repository instructions                                                                              |
| `/.cursorrules`                    | Cursor repository rules                                                                                             |
| `/skill-packages/`                 | Reusable multi-agent skill packages                                                                                 |

Use `/llms.txt` for quick retrieval and `/llms-full.txt` when an assistant needs a self-contained prompt payload.

## What To Give An Agent

| Task                                       | Include                                                                                |
| ------------------------------------------ | -------------------------------------------------------------------------------------- |
| Build a React canister integration         | `/llms-full.txt` and the generated `idlFactory` or Candid service                      |
| Refactor hooks or cache invalidation       | `/skill-packages/ic-reactor-hooks/SKILL.md`                                            |
| Change package exports or generated output | `/skill-packages/ic-reactor-packages/SKILL.md`                                         |
| Add auth or identity attributes            | `packages/react/src/auth/`, `packages/react/README.md`, and the relevant app setup |
| Configure generation                       | `packages/cli/README.md` or `packages/vite-plugin/README.md`                           |

## Agent Skills

The in-repo skills are:

- `ic-reactor-hooks`: React hook factories, generated hooks, query/mutation factories, cache invalidation, and inside-vs-outside React usage
- `ic-reactor-packages`: package ownership, dependency boundaries, generated files, exports, and verification workflow

The hook skill is also available externally:

- **In-repo**: [`skill-packages/ic-reactor-hooks/`](https://github.com/B3Pay/ic-reactor/tree/main/skill-packages/ic-reactor-hooks) — used by agents working directly in this repository
- **External**: [`B3Pay/ic-reactor-skills`](https://github.com/B3Pay/ic-reactor-skills) — standalone installable skill for use in any ICP project

Use this skill when asking an AI agent to generate or refactor IC Reactor integrations, especially when you need:

- `createActorHooks(...)` patterns
- reusable `createQuery` / `createMutation` factory modules
- guidance for using the same query or mutation objects inside React and outside React
- cache invalidation wiring with `getQueryKey()` / `invalidate()`
- guidance on when to use manual hooks vs generated hooks (CLI / Vite plugin)

### Example Prompt

```text
Use $ic-reactor-hooks to refactor this canister integration into reusable query and mutation factories, then show usage in a React component and a loader/action.
```

### Example Install (`skills` CLI)

```bash
npx skills add B3Pay/ic-reactor-skills --full-depth --skill ic-reactor-hooks
```

## Prompting Checklist

When asking an AI assistant to write IC Reactor code, include:

- the target package or app path
- the Candid service or generated `idlFactory` and `_SERVICE` type
- whether the code runs inside React, outside React, or both
- whether UI-friendly display values (`DisplayReactor` / `display: true`) are desired
- whether the project should use generated hooks, manual hooks, or reusable factories
- the verification command you expect the agent to run

## Inside React vs Outside React

This is the most important distinction for generated or AI-written integrations:

- React hooks (`useActorQuery`, `.useQuery()`, `.useMutation()`, etc.) belong only in React components or custom hooks.
- Route loaders, actions, services, scripts, and tests should use imperative APIs such as `.fetch()`, `.prefetch()`, `.execute()`, `.invalidate()`, `.getCacheData()`, or direct reactor methods.

## Prompting Examples

### Setup

```text
Use llms-full.txt and set up a display reactor for canister backend using the IDL factory from ./declarations/backend. Show a React component and the verification command.
```

### Shared Factories

```text
Use $ic-reactor-hooks to refactor this canister integration into reusable query and mutation factories. Show usage inside a React component and in a route loader/action.
```

### Package Changes

```text
Use $ic-reactor-packages to review this codegen change. Identify the package owner, generated-file boundaries, affected docs, and focused tests.
```

## Generated Files

Agents should not edit generated declarations or generated hook files by hand. If generated output is wrong, update the source `.did`, wrapper module, or generator under `packages/codegen/src/`, then regenerate intentionally.

For repository work, generated/transient paths include `dist/`, `.dfx/`, `.icp/`, `.mops/`, `target/`, `.next/`, `.astro/`, `*.tsbuildinfo`, generated canister declarations, and generated hook files.

## Tools and Extensions

If you use **Cursor**, this project includes a `.cursorrules` file that automatically configures the AI to follow the project's specific coding standards and architecture.

If you use **GitHub Copilot**, the repository includes `.github/copilot-instructions.md` and skill metadata at `skill-packages/ic-reactor-hooks/agents/copilot.yaml`. You can also install the external skills from [`B3Pay/ic-reactor-skills`](https://github.com/B3Pay/ic-reactor-skills).

If you use **Claude**, the repository includes a `CLAUDE.md` project context file and skill metadata at `skill-packages/ic-reactor-hooks/agents/claude.yaml`.

If you use **OpenAI Codex**, the repository includes `AGENTS.md` and skill metadata at `skill-packages/ic-reactor-hooks/agents/openai.yaml`.
