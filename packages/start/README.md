# @ic-reactor/start

> React/TanStack app starter for **fully on-chain** Internet Computer frontends.

`@ic-reactor/start` is the fastest path to a React app that deploys to an ICP
asset canister. It composes the existing IC Reactor packages with
[TanStack Router](https://tanstack.com/router) and [`icp-cli`](https://github.com/dfinity/icp-cli):

- **`icReactorStart`** â€” a Vite preset that wires up `@ic-reactor/vite-plugin`
  (Candid codegen + `ic_env` cookie injection) and TanStack Router file routes.
- **`defineIcReactorStartConfig`** â€” typed config helper.
- **`createApp`** â€” the scaffold that writes a complete V0 app to disk.

> **V0 scope:** CSR/static only, deployed to an ICP asset canister. No SSR, no
> TanStack Start server functions, no colocated backend functions. SSR and
> on-chain SEO rendering are on the roadmap â€” see
> [`docs/plans/ic-reactor-start-v0.md`](../../docs/plans/ic-reactor-start-v0.md).
> This is **not** a TanStack Start clone; it is the React/TanStack starter for
> ICP frontends.

## Create an app

```bash
pnpm create ic-reactor-start my-app
# or:
ic-reactor create start my-app
```

Then:

```bash
cd my-app
pnpm install
icp network start -d   # start a local replica with Internet Identity
icp deploy             # deploy backend + frontend canisters
pnpm dev               # start Vite
```

Canister IDs are resolved at runtime through `icp-cli` and the `ic_env` cookie
injected by `@ic-reactor/vite-plugin`. **There is no `.env` file for canister
IDs.**

## Generated layout

```text
my-app/
  icp.yaml              # icp-cli: backend + frontend canisters, managed local net
  ic-reactor.json       # Candid codegen config
  backend/
    canister.yaml       # Motoko recipe (V0 starter)
    main.mo             # counter sample canister
    backend.did
  frontend/
    canister.yaml       # Vite build + asset sync
    package.json
    vite.config.ts      # uses icReactorStart()
    index.html
    src/
      main.tsx          # TanStack Router + QueryClient bootstrap
      routeTree.gen.ts  # regenerated by the router plugin
      routes/__root.tsx
      routes/index.tsx  # backend query/update + II login sample
      lib/client.ts     # queryClient + clientManager
      lib/auth.ts       # AuthenticationManager + auth hooks
```

## The Vite preset

```ts
// frontend/vite.config.ts
import { defineConfig } from "vite"
import react from "@vitejs/plugin-react"
import { icReactorStart } from "@ic-reactor/start/plugin/vite"

export default defineConfig({
  plugins: [
    ...icReactorStart({
      // object form (recommended) or CanisterConfig[]
      canisters: {
        backend: { didFile: "../backend/backend.did" },
      },
    }),
    react(),
  ],
})
```

The preset:

- Passes normalized canister config into `@ic-reactor/vite-plugin` (the sole
  owner of codegen + `ic_env` injection).
- Includes TanStack Router plugin defaults (`target: "react"`, auto code-split).
- Proxies local `/api` to the `icp-cli` local gateway/replica.
- Honors `ICP_ENVIRONMENT`.
- Never hard-codes canister IDs.

Options:

| Option              | Type                               | Default            | Description                                            |
| ------------------- | ---------------------------------- | ------------------ | ------------------------------------------------------ |
| `canisters`         | `object \| CanisterConfig[]`       | â€”                  | Canister declarations (object keyed by name, or array) |
| `outDir`            | `string`                           | `src/declarations` | Output dir for generated declarations                  |
| `clientManagerPath` | `string`                           | `../../lib/client` | Import path to shared `clientManager`                  |
| `target`            | `"react" \| "core"`                | `"react"`          | Generated runtime target                               |
| `injectEnvironment` | `boolean`                          | `true`             | Inject `ic_env` cookie for local dev                   |
| `router`            | `boolean \| { autoCodeSplitting }` | `true`             | Include TanStack Router plugin                         |
| `proxyApi`          | `boolean`                          | `true`             | Proxy `/api` to local replica                          |

```ts
  force: false,

```

## Deploy

Local:

```bash
icp deploy
icp sync
```

Mainnet:

```bash
icp deploy -e ic
```

## Why V0 is CSR/static

V0 optimizes for the fastest credible path to a working, fully on-chain app.
The asset canister serves static files, the browser resolves canister IDs via
the `ic_env` cookie, and the app renders client-side. SSR/server functions are
future work.

**On-chain SEO rendering** (serving dynamic HTML through a canister
`http_request`) is the ICP-native long-term path for SEO, tracked as V2. It is
deliberately separate from generic Node SSR.
